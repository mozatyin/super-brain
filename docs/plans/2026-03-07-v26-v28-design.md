# V2.6-V2.8 Design: Soul-Informed Detection + Quality Control + Ensemble

## Overview

Three focused iterations to improve detection accuracy by leveraging the rich Soul state built in V2.4-V2.5.

| Version | Focus | Hypothesis |
|---------|-------|------------|
| V2.6 | Soul-Informed Detection | Passing Soul context to Detector improves calibration |
| V2.7 | ThinkDeep Quality Control | Reducing over-extraction noise improves signal quality |
| V2.8 | ThinkSlow Trajectory Ensemble | Blending incremental estimates with final detection reduces variance |

## Current State (V2.5)

- 20t MAE: 0.178 (best ever)
- 10t MAE: 0.185 (regressed from V2.4's 0.164)
- Soul: 62.7 facts, 34.7 secrets, 31.3 intentions, 28.3 gaps per profile
- Over-extraction: 31 intentions per profile is noise, not signal
- Detector blind spot: doesn't know about Soul state at detection time

---

## V2.6: Soul-Informed Detection

### Problem

Detector runs 7 batch LLM calls with zero knowledge of the Soul state (62 facts, reality snapshot, intentions). This context could significantly improve calibration — e.g., knowing "works in tech" prevents inflating need_for_cognition.

### Design

Add optional `soul_context` to Detector.analyze(), injected into each batch's user_message as a `## Background Information About This Person` section.

**What to include** (concise, not overwhelming):
- Top 10 facts by confidence
- Reality summary + constraints + resources
- Top 3 intentions by strength

**What NOT to include** (avoid biasing):
- Secrets, contradictions, gaps (could anchor Detector to specific narratives)

**Prompt framing** (critical — context not instruction):
```
## Background Information About This Person
The following facts were gathered from earlier in the conversation:
- [career] Software engineer at a startup (high confidence)
- [relationship] Single, values independence
...
Reality: Currently a mid-career engineer feeling stuck...
Intentions: Wants to start own business (strong), wants to travel more (moderate)

Use this as CONTEXT to calibrate your scoring, but base scores on OBSERVED
BEHAVIOR in the text, not on these facts alone.
```

### Changes

1. `super_brain/detector.py` — `Detector.analyze()` gets `soul_context: str | None = None`, injected into each batch user_message
2. `eval_conversation.py` — `detect_and_compare()` gets `soul: Soul | None = None`, builds soul context string
3. `eval_conversation.py` — `run_eval()` passes soul to detect_and_compare()
4. New helper: `_build_detector_soul_context(soul: Soul) -> str` in detector.py or eval_conversation.py

### Expected Impact

- 20t MAE: 0.178 → ~0.165 (COG, NEU, SOC improvement)
- 10t MAE: likely unchanged (Soul not rich enough at 10 turns)

## V2.7: ThinkDeep Quality Control + Dedup

### Problem

- 31 intentions, 28 gaps per profile — over-extraction creates noise
- `td_fired` resets after push mode, allowing unlimited re-triggering
- 34.7 secrets per profile — many are near-duplicates
- FactExtractor contradictions (30.0 avg) also likely inflated

### Design

**1. ThinkDeep trigger cap**: Max 2 fires per conversation.
- Replace `td_fired: bool` with `td_fire_count: int`
- Check `td_fire_count < 2` instead of `not td_fired`
- After push mode clears `last_td`, increment count (don't reset)

**2. Text deduplication module**: `super_brain/dedup.py`
- `is_duplicate(new_text: str, existing_texts: list[str], threshold: float = 0.6) -> bool`
- Algorithm: token set overlap ratio (Jaccard-like)
- `tokens(text) = set(text.lower().split())`
- `overlap = len(a & b) / len(a | b)`
- If overlap > threshold → duplicate

**3. Apply dedup at accumulation points**:
- `soul.intentions.extend()` → filter duplicates first
- `soul.gaps.extend()` → filter by bridge_question similarity
- `soul.secrets.extend()` → filter duplicates
- `soul.contradictions.extend()` → filter duplicates
- `soul.facts.extend()` → already has dedup in FactExtractor

### Changes

1. `super_brain/dedup.py` — New module with `is_duplicate()` and `dedup_list()`
2. `eval_conversation.py` simulate_conversation — td_fire_count cap, dedup calls
3. Tests for dedup module

### Expected Impact

- Intentions per profile: 31 → ~6-8
- Gaps per profile: 28 → ~4-6
- Secrets per profile: 34 → ~10-15
- Cleaner Soul state → better V2.6 detector context
- 10t MAE may recover (less push mode disruption)

## V2.8: ThinkSlow Trajectory Ensemble

### Problem

ThinkSlow runs 9 times producing progressive trait estimates with confidence scores. But final detection uses only the Detector's one-shot result, wasting the incremental learning.

### Design

**Ensemble algorithm**: Final score = weighted blend of Detector + ThinkSlow trajectory.

```python
for each trait:
    ts_values = [ThinkSlow estimates for this trait across all cycles]
    ts_confs  = [corresponding confidence scores]

    if ts_values:
        ts_avg = weighted_mean(ts_values, weights=ts_confs)
        ts_weight = mean(ts_confs) * 0.4  # ThinkSlow max 40% contribution
        det_weight = 1.0 - ts_weight
        final = det_weight * detector_value + ts_weight * ts_avg
    else:
        final = detector_value  # no ThinkSlow data → pure Detector
```

**Key design choices**:
- ThinkSlow max weight: 40% (Detector sees full text, more reliable)
- Weight scales with confidence: low-confidence ThinkSlow estimates contribute less
- Traits never estimated by ThinkSlow: pure Detector (no penalty)

### Changes

1. `super_brain/ensemble.py` — New module: `blend_with_trajectory(detected: PersonalityDNA, ts_results: list[ThinkSlowResult]) -> PersonalityDNA`
2. `eval_conversation.py` detect_and_compare() — accepts `ts_results`, calls blend after detection
3. `eval_conversation.py` run_eval() — passes ts_results
4. Tests for ensemble module

### Expected Impact

- Reduced per-dimension variance (fewer catastrophic errors)
- ≤0.40 rate: 91.9% → ~95%+
- 20t MAE: further improvement from ensemble smoothing
